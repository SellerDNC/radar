📋 Decisões Técnicas Registradas
DecisãoEscolha feitaMotivo
OAuth ML bloqueado por certificaçãoWorkaround: busca pública para MercadoX, OAuth para Dashboard/RankingML exige certificação para endpoints privados — contornar com dados públicos onde possível
Multi-tenant Fase 1tenant_id gerado no browser (localStorage), sem autenticaçãoUso pessoal — sem risco · Fase 2 terá Supabase Auth
API Key AnthropicUsuário passa via header X-API-Key em cada requestNão armazenar no Worker · Segurança · Cada usuário usa a própria chave
Dashboard — eliminar XLSXSync direto ML API → D1 via Worker, paginado 500 pedidos/chamadaDados reais, atualizados, sem friction de importação manual
Frontend Fase 1Single HTML file — sem build, sem frameworkDeploy trivial, validação rápida, fácil de manter em Fase 1
StorageD1 (SQL) para histórico · KV para tokens OAuthD1 permite queries complexas (ABC, KPIs) · KV é ideal para key-value simples e rápido

🏆 Diferencial Competitivo
O que o Radar tem
✓Análise de mercado ML com critérios objetivos (C1 + C2)
✓Dashboard direto da API ML (sem planilha)
✓Ranking de anúncios com histórico e gráfico
✓Diagnóstico automatizado de anúncio (score + IA)
✓Ficha técnica gerada por IA
✓Calendário de rotina personalizado por IA
✓Agente IA com contexto real da operação
✓Histórico persistente em banco de dados
✓Tudo integrado em um só lugar
Concorrentes não têm
✗Análise de oportunidade + financeiro integrados
✗IA que conhece a operação real do seller
✗Calendário de rotina gerado com contexto próprio
✗Diagnóstico algorítmico + IA no mesmo lugar
✗Foto → IA identifica o produto automaticamente
✗Stack serverless — custo quase zero em Fase 1
✗Construído por quem vende no ML de verdade
